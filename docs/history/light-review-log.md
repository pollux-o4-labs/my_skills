# 경량 심사 실적 로그 (light-review-log)

기존 스킬 소규모 수정 PR 을 풀 파이프라인(skill-review.js) 대신 약식(적대 리뷰어 1기 + 세션 직접 수정)으로 심사한 실적의 적립 장부. **규정 아님** — 매 건 재량 판단이며, 규정화는 **5건 이상 적립 + 사후 결함 0** 도달 시 재상정한다(2026-07-15 사용자 결정: 성급한 규정화 반려).

## 판정 인덱스 — 규정화 판단은 이 절만 읽는다

형식: `- #PR 스킬 — 잡음(약식이 걸러낸 결함) / 생략 리스크(안 돌린 것 중 물릴 수 있는 것) / 사후: 상태 / 결과: 상태`.
- **사후 칸이 판정 재료**(수동): 이후 세션에서 해당 스킬의 결함·미발동·오발동 발견 시 이 줄을 갱신하고, 원인이 생략 리스크면 `실패`로 계상. 5건 도달 시 사후 칸에 `실패` 0 이면 규정화 상정.
- **결과 칸은 자동**: `.github/workflows/ledger-merge-status.yml` 이 PR close 시 `머지됨(sha, 날짜)`·`닫힘·불채택(날짜)` 으로 덮어쓴다(사람 결정을 받아적기만). Land 단계는 이 필드를 `대기` 로 미리 써 두는 것을 권장한다(마커가 없어도 Action 이 줄 끝에 붙이지만, 시드가 있어야 형식이 일관). 손으로 고치지 말 것.
