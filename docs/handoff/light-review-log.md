# 경량 심사 실적 로그 (light-review-log)

기존 스킬 소규모 수정 PR 을 풀 파이프라인(skill-review.js) 대신 약식(적대 리뷰어 1기 + 세션 직접 수정)으로 심사한 실적의 적립 장부. **규정 아님** — 매 건 재량 판단이며, 규정화는 **5건 이상 적립 + 사후 결함 0** 도달 시 재상정한다(2026-07-15 사용자 결정: 성급한 규정화 반려).

## 기재 규약 — 평가 자체의 품질을 후속 세션이 검증할 수 있어야 한다

매 건 아래 7항을 빠짐없이 기재한다. 특히 ③ 생략 단계와 ⑥ 자기반증은 생략 금지 — 생략을 숨기면 "다 검사했다"로 읽힌다.

1. **대상** — PR 번호, 스킬, diff 규모
2. **경로 선택 근거** — 왜 풀 파이프라인이 과투자인가
3. **생략 단계 명시** — 프로브(무보정)·replay·파생가능성 프로브·fix⇄verify 루프 중 무엇을 안 돌렸나
4. **투입 구성** — 리뷰어 수·모델·공격 초점 수
5. **결과** — BLOCKER/CHEAP/NOISE 건수, 수정 커밋, 머지 여부
6. **자기반증** — "풀 파이프라인이었으면 추가로 잡았을 결함은 무엇인가" 한 줄 판단
7. **사후 판정** — 기록 시점엔 `미정`. 이후 세션에서 해당 스킬에 결함·미발동·오발동이 발견되면 이 줄을 갱신하고, 원인이 ③의 생략 단계면 **경량 경로 실패로 계상**한다. 5건 도달 시점에 이 항들이 판정 근거가 된다.
