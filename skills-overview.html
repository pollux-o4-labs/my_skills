Karpathy Guidelines

multica-ai/andrej-karpathy-skills ↗

Andrej Karpathy가 관찰한 LLM 코딩 실수 패턴에서 도출된 4가지 행동 원칙.
Claude 자체 행동을 교정하는 가드레일 — 레포에 설치하면 항상 적용.

$ npx skills@latest add multica-ai/andrej-karpathy-skills -g
01

Think Before Coding

가정을 숨기지 말고 먼저 드러낸다. 해석이 여러 개면 골라서 진행하지 않고 사용자에게 제시한다.

02

Simplicity First

요청한 것만 구현. 200줄이 50줄이 될 수 있으면 다시 짠다. 추측성 추상화, 미래 대비 코드 금지.

03

Surgical Changes

내가 바꾼 것만 건드린다. 주변 코드 개선, 무관한 리팩터링 금지. 모든 변경 라인은 요청에서 역추적 가능해야 한다.

04

Goal-Driven Execution

검증 기준을 먼저 정의하고 루프를 돈다. "작동하게 해줘"가 아니라 "이 테스트를 통과시켜줘".

Matt Pocock Skills

mattpocock/skills ↗

레포 단위 에이전트 워크플로우 세팅. 이슈 트래커·트리아지·TDD·아키텍처 개선까지 커버.
/setup-matt-pocock-skills 한 번 실행 후 아래 스킬들이 레포 컨텍스트를 자동 인식.

$ npx skills@latest add mattpocock/skills -g
스킬 설명
/setup-matt-pocock-skills
선행 필수 		이슈 트래커(GitHub / GitLab / 로컬 마크다운), 트리아지 라벨 어휘, 도메인 문서 위치를 레포에 등록. 나머지 스킬 쓰기 전 한 번만 실행.
/to-prd
 대화 맥락을 PRD로 변환해 이슈 트래커에 게시. 유저 인터뷰 없이 현재 컨텍스트에서 바로 합성.
/to-issues
 PRD나 계획을 독립 실행 가능한 이슈 단위로 분해. 수직 슬라이스(트레이서 불릿) 방식으로 나눠 에이전트가 바로 집어갈 수 있게.
/triage
 들어온 이슈를 상태 머신으로 분류. needs-triage → needs-info → ready-for-agent → ready-for-human → wontfix 흐름.
/diagnose
 재현 → 최소화 → 가설 → 계측 → 수정 → 회귀 테스트 순서로 버그 진단. 빠른 피드백 루프 구축이 핵심.
/tdd
 Red-Green-Refactor 루프. 공개 인터페이스 기반 통합 테스트 우선, 구현 내부를 직접 테스트하는 안티패턴 방지.
/improve-codebase-architecture
 얕은 모듈을 깊은 모듈로 리팩터링할 포인트를 찾아줌. 테스트 가능성과 AI 탐색성 향상이 목표.
/zoom-out
 모르는 코드 영역에서 한 레이어 위로 올라가 관련 모듈 전체 맵을 보여줌.

Karpathy는 Claude 자체 행동 교정 / Matt Pocock은 레포별 워크플로우 연결 — 같이 쓰면 시너지.
